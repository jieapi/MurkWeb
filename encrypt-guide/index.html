在线加密

MurkPHP 加密说明

MurkPHP 在线加密服务用于对用户上传的 PHP 文件或 ZIP 源码包进行加密处理。提交任务前，请确认代码来源合法、文件内容完整，并根据项目实际运行环境选择合适的加密算法和参数。

使用前确认

加密前代码应先在目标环境中正常运行

平台标注的 PHP 7.0 至 PHP 8.4 兼容范围，是指加密产物可在对应 PHP 版本环境中运行的支持范围。它以加密前代码本身已经支持目标 PHP 版本为前提。

文件类型

支持上传 PHP 文件和 ZIP 源码包。ZIP 包适合一次性处理多个 PHP 文件。

兼容范围

加密产物支持 PHP 7.0 至 PHP 8.4 环境运行，前提是原始代码本身适配目标版本。

结果下载

任务成功后可下载加密结果包，结果包包含处理后的文件和日志信息。

文件安全说明

上传源码只用于本次加密处理

平台不会主动泄露、传播或用于非服务目的处理用户上传的源码文件。为降低源码长期留存风险，系统会按任务状态自动清理上传源文件和加密结果文件。

用途限定

上传的源码文件仅用于执行本次在线加密任务，平台不会主动传播或用于无关用途。

敏感源码建议

如项目涉及严格保密义务、商业核心资产或不适合交给第三方处理的内容，建议不要上传到平台。

自动清理

未提交任务的源文件会自动清理；已提交任务的源文件会在任务完成后清理，加密结果文件保留 30 天。

兼容性说明

自动降级能力只适合辅助处理部分语法差异

自动降级不能保证覆盖所有语言特性、框架行为和业务代码场景。启用后应在目标 PHP 版本环境中完整测试主要功能、异常分支、依赖加载和接口调用结果。

先测试原始代码

如果加密前代码在目标环境中无法正常运行，加密后通常也不能解决运行环境或依赖兼容问题。

谨慎使用激进模式

激进模式适合在充分测试后使用，不建议直接用于生产项目的最终交付版本。

目标环境验证

启用自动降级后，请在目标 PHP 版本环境中测试主要功能、异常分支和接口调用结果。
